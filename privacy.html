Application Permissions

Our application requires certain permissions to function properly, and we believe in complete transparency regarding why each permission is necessary:

  2. Network State Access Permission: We require this permission to monitor the status of your device's internet connection.   3. Vibrate Permission: This permission is used to provide haptic feedback during gameplay, enhancing the user experience with tactile responses to various game events. This feature is entirely optional and can be disabled in the application settings if desired.

  4. Wake Lock Permission: To prevent interruptions during crucial gameplay moments, we implement the wake lock permission. This ensures your device's screen remains active during gameplay sessions, preventing automatic screen timeout which could disrupt your gaming experience.

  5. Billing Permission: This permission is essential for processing in-app purchases through the Google Play Billing system. It allows us to offer optional premium features, virtual currency, and cosmetic items while ensuring secure and reliable transaction processing.
